# Contributing

## Workflow
- Create working branches from `develop`
- Use the approved branch naming policy
- Open pull requests into `develop`
- Do not push directly to `main` or `develop`

## Allowed branch prefixes
- `feat/*`
- `fix/*`
- `refactor/*`
- `docs/*`
- `chore/*`
- `test/*`
- `ci/*`
- `release/*`
- `hotfix/*`

## Pull requests
- Keep pull requests focused
- Avoid unrelated changes
- Update documentation when needed
- Make sure changes match the current project stage

## Notes
- Version-sensitive implementation decisions must be verified against current documentation first
- Frontend CI checks should be added when frontend setup becomes active
